# Админка Student's Life

## Общий порядок работы

1. Войдите в `/admin/` под администратором.
2. Слева используйте постоянное меню: доступ, заявки/чаты, контент, каталог.
3. Сначала заполните справочники: страны, города, офисы, услуги.
4. Затем добавляйте вузы и программы.
5. После этого можно публиковать баннеры, новости, базу знаний и принимать заявки.

## Доступ и пользователи

| Раздел | Для чего нужен | Как заполнять |
| --- | --- | --- |
| Пользователи | Аккаунты для входа в админку и приложение | Нажмите "Добавить пользователя", укажите username/email, пароль, имя, фамилию, `is_active`. `is_staff` включайте только тем, кто должен входить в Django admin. |
| Профили клиентов | Данные клиентов мобильного приложения | Обычно создаются автоматически при регистрации. Вручную выберите пользователя, роль `user`, телефон, WhatsApp/Telegram, страну, город, гражданство. |
| Роли приложения | `user` и `manager` для mobile API | `user` не даёт менеджерские права. У `manager` должен быть включён флаг `Manager access`. |
| Сотрудники | Менеджеры Student's Life | Сначала создайте пользователя, затем добавьте сотрудника и выберите этого пользователя. Один пользователь может быть связан только с одним сотрудником. |
| Активность | Online/last seen | Заполняется приложением автоматически. Используется для диагностики. |

## Заявки и чаты

| Раздел | Для чего нужен | Важные поля |
| --- | --- | --- |
| Заявки | Заявки из мобильного приложения | ФИО, телефон, WhatsApp, Telegram, email, страна/город/вуз/программа, услуга, статус, комментарий. |
| Файлы заявок | Документы клиента | Файл, тип файла, исходное имя, кто загрузил. |
| История статусов | Аудит изменения статуса | Создаётся при изменениях статуса, вручную обычно не заполняется. |
| Чаты | Комнаты общения клиента и менеджера | Пользователь, заявка, назначенный менеджер, статус. |
| Сообщения | Текстовые сообщения | Комната, отправитель, текст, тип сообщения, прочитано/не прочитано. |
| Вложения чата | Фото в чате | Создаются через API при загрузке изображения. |

## Контент приложения

| Раздел | Где видно в приложении | Как заполнять |
| --- | --- | --- |
| Услуги | Экран "Услуги", форма заявки | Название, slug, краткое описание, полное описание, изображение, иконка, порядок сортировки, активность. |
| Новости | Экран новостей | Заголовок, slug, краткое описание, текст, изображение, категория, дата публикации, активность. |
| База знаний | Раздел материалов | Заголовок, slug, описание, текст, категория, теги, изображение, публикация. |
| Баннеры главной | Главный экран | Slot `hero` или `news`, заголовок, описание, изображение, CTA, связанная новость/услуга/вуз. |
| Контакты офисов | Главная/контакты | Страна, город, офис, адрес, телефон, WhatsApp, Telegram, email, соцсети, активность. |
| Политика конфиденциальности | `/privacy-policy/` и настройки | Держите одну активную запись. Если активной нет, показывается текст по умолчанию. |

## Каталог образования

| Раздел | Связи | Обязательные поля |
| --- | --- | --- |
| Страны | Страна -> города -> вузы | Название, slug, описание, флаг/обложка, активность. |
| Города | Город принадлежит стране | Страна, название, slug, описание, фото, расходы. |
| Офисы | Офис может быть связан со страной/городом | Название, адрес, телефон/WhatsApp/Telegram/email, график, активность. |
| Вузы | Вуз связан со страной и городом | Название, slug, страна, город, описание, логотип/обложка, сайт, языки, уровни, общежитие, стоимость. |
| Программы | Программа принадлежит вузу | Вуз, название, уровень, факультет, специальность, язык, длительность, стоимость, дедлайн, документы, требования. |

## Рекомендуемый порядок заполнения каталога

1. Создайте страну.
2. Создайте города этой страны.
3. Создайте офисы и контакты.
4. Создайте вуз, выбрав страну и город.
5. Внутри вуза или в разделе "Программы" добавьте программы.
6. Проверьте, что записи активны (`is_active`) и имеют понятный `sort_order`.

## Менеджеры в мобильном приложении

Менеджер входит в приложении через кнопку "Войти как менеджер" в настройках. Сервер проверяет локальный логин и пароль Django и разрешает вход только пользователям, у которых в админке создан активный профиль `Staff`. После успешного входа сервер выдаёт роль `manager`, и сотрудник видит клиентские заявки и чаты в мобильном приложении.

## Частые ошибки

| Симптом | Что проверить |
| --- | --- |
| Пользователь не видит заявки | У профиля роль `user`, заявка должна быть связана с этим пользователем. |
| Менеджер не видит чаты | У профиля должна быть роль `manager` с включённым `Manager access`. |
| Нельзя создать сотрудника | Сначала создайте пользователя. Один пользователь не может быть у двух сотрудников. |
| В приложении нет изображения | Проверьте, что файл загружен, media отдаётся сервером, а запись активна. |
| Пункт меню не виден | Проверьте права пользователя в Django admin и `is_staff`. |
